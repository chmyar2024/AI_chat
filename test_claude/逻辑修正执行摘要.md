# 模板逻辑问题诊断与修正方案 - 执行摘要

## 🔍 核心问题

### 主要逻辑缺陷
1. **顺序混乱**: 没有遵循"定义→目标→方法→追踪→反馈"的自然流程
2. **层级过深**: 很多模板有10+个一级章节，3-4级子标题
3. **工具位置**: 记忆工具、自动化机制散落在中间
4. **时间线模糊**: 缺少清晰的过去→现在→未来结构
5. **实用性不足**: 缺少"快速参考"和"检查清单"

### 典型错误案例

**00_Daily**:
- ❌ "认知仪表盘"在"今日概览"之前（应该先概览再深入）
- ❌ "记忆训练"插在中间（应该在晨间或晚间）
- ❌ 缺少清晰的早→日→晚时间线

**02_Action**:
- ❌ "为什么现在"在"完成标准"之后（应该在定义阶段）
- ❌ 执行、能量、风险顺序混乱

**06_Insight**:
- ❌ "记忆宫殿"在"结构化表达"之后（应该在最后工具区）
- ❌ "间隔重复"插在中间（应该在最后）
- ❌ "配对联想"与"记忆宫殿"内容重复

**23_Book**:
- ❌ 章节过多（10+个），层级混乱
- ❌ "行动转化"太晚出现
- ❌ 记忆工具散落各处

**26_Skill**:
- ❌ 章节过多（15+个）
- ❌ "学习路径"和"刻意练习"应该合并
- ❌ "教学"和"变现"不应该是核心章节

## ✅ 统一修正方案

### 标准逻辑框架

```markdown
[元信息区]
├─ YAML元数据
├─ 核心原则说明
└─ 边界说明（如有）

[主体内容] (4-6个一级章节)
├─ 1️⃣ 定义与现状 (What & Where)
├─ 2️⃣ 价值与目标 (Why & Goal)
├─ 3️⃣ 方法与执行 (How & Do)
├─ 4️⃣ 连接与关系 (Relation)
├─ 5️⃣ 追踪与演化 (Track)
└─ 6️⃣ 反馈与优化 (Feedback)

[实用工具区]
├─ ⚡ 快速参考 (3-5条关键信息)
├─ ⚡ 检查清单 (质量标准)
└─ ⚡ 使用提示 (频率/时长)
```

### 按类型的具体结构

**输入型** (Seed, Intuition, Daily):
→ 捕捉 → 反应 → 连接 → 处理

**加工型** (Insight, Action, Project, Skill):
→ 定义 → 现状 → 执行 → 资源 → 追踪 → 反馈

**关系型** (Persona, Organization, Community):
→ 画像 → 关系 → 价值 → 互动 → 维护 → 发展

**反馈型** (Loop, Meta, Recall):
→ 事实 → 分析 → 学习 → 更新

**战略型** (Vision, Heart, Scale):
→ 愿景 → 现状 → 路径 → 连接 → 校准
