> [!NOTE] 核心原则
> **目的**: 生成新组合、新想法、新解决方案
> **触发**: 需要解决新问题或创造新价值
> **规则**: 先求数量，后求质量；先发散，后收敛

> [!WARNING] Create vs Play 边界
> **用Create**: 当有明确的产出目标（必须预设[[04_Asset/XX]]）
> **用Play**: 当需要纯粹探索/恢复能量（允许无产出）
> 
> **关键区别**:
> - Create = 目标导向 + 结构化流程 + 必须产出Asset
> - Play = 过程导向 + 自由探索 + 允许零产出
> 
> **判断标准**: 问"如果什么也没做出来，我会失望吗？"
> - 如果会失望 → Create（你需要产出）
> - 如果不会失望 → Play（你需要过程）
