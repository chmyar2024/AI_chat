# 🧭 模板选择决策树

> 不知道该用哪个模板？这个指南帮你快速选择！

---

## 📥 我想记录外部信息

### 信号捕捉
**来源可追溯** (文章/对话/观察) → [[01_Seed]]  
**莫名的感觉** (预感/直觉/"我就是知道") → [[10_Intuition]]

**快速判断**: 问自己"我是从哪里知道这个的？"
- 能说清楚来源 → Seed
- 说不清为什么知道 → Intuition

---

## 🧠 我想记录认知/想法

### 单个洞察
**一个独立的理解/原则** → [[06_Insight]]

### 知识体系
**完整的领域知识** (多个Insight的集合) → [[07_System]]

### 读书笔记
**深度阅读一本书** → [[23_Book]]

### 学习记录
**结构化课程学习** (在线课程/训练营) → 暂无专门模板，用 [[23_Book]] 或 [[07_System]]

**快速判断**:
- 单个想法 → Insight
- 一本书 → Book
- 整个领域 → System

---

## ⚡ 我想记录行动/执行

### 任务管理
**单个任务** (天/周级别) → [[02_Action]]  
**多任务容器** (月/季度级别) → [[03_Project]]

### 创造vs玩耍
**有产出目标** (必须做出Asset) → [[09_Create]]  
**纯粹探索/恢复能量** (允许无产出) → [[17_Play]]

**快速判断**: 问"如果什么也没做出来，我会失望吗？"
- 会失望 → Create (你需要产出)
- 不失望 → Play (你需要过程)

## 👥 我想记录关系/实体

### 个人关系
**具体的某个人** (1对1关系) → [[05_Persona]]

### 组织机构
**正式组织** (公司/政府/学校/NGO) + 有明确结构 → [[24_Organization]]

### 群体社群
**非正式群体** (读书会/朋友圈/兴趣小组) + 无固定结构 → [[25_Community]]

**快速判断**:
- 单个人 → Persona
- 公司/机构 (有法律实体) → Organization
- 兴趣小组/朋友圈 → Community

## 🔍 常见困惑

### 困惑1: Seed vs Intuition
❓ 我读文章时突然"感觉"这个很重要，用哪个？  
✅ **Seed** - 因为来源是文章（外部），即使你说不清为什么重要

❓ 我做梦/冥想时突然"知道"了什么，用哪个？  
✅ **Intuition** - 因为来源是内在，说不清为什么

### 困惑2: Create vs Play
❓ 我在玩乐高但想做个特定的模型，用哪个？  
✅ **Create** - 因为有明确产出目标

❓ 我在随便画画放松，用哪个？  
✅ **Play** - 因为目的是过程，不是产出

### 困惑3: Skill vs System
❓ 我在学编程，用哪个？  
✅ **Skill** - 追踪练习和熟练度  
✅ **System** - 理解计算机科学理论  
💡 **两个都要！** 技能和知识分开管理

❓ 我在学习某个理论，用哪个？  
✅ **System** - 理论是"知道"，不是"会做"

### 困惑4: Persona vs Organization vs Community
❓ 我在和Google公司合作，用哪个？  
✅ **Organization** - Google是正式组织  
✅ **Persona** - Google的具体联系人

❓ 我参加读书会，用哪个？  
✅ **Community** - 读书会是非正式群体  
✅ **Persona** - 读书会的具体成员

### 困惑5: Book vs Insight
❓ 读书时创建的洞察放哪里？  
✅ **两个地方都要**:
- Book中: 只保留链接 + 一句话摘要
- Insight中: 保留完整详细内容
💡 **不要在Book中重复写Insight的完整内容！**

## ⚡ 快速决策流程

```
开始
  ↓
问题1: 这是外部信息吗？
  ├─ 是 → 能说清来源吗？
  │        ├─ 能 → [[01_Seed]]
  │        └─ 不能 → [[10_Intuition]]
  │
  └─ 否 → 问题2: 这是行动吗？
           ├─ 是 → 有产出目标吗？
           │        ├─ 有 → 单任务还是项目？
           │        │      ├─ 单任务 → [[02_Action]]
           │        │      └─ 项目 → [[03_Project]]
           │        └─ 无 → [[17_Play]]
           │
           └─ 否 → 问题3: 这是认知吗？
                    ├─ 是 → 单个还是体系？
                    │      ├─ 单个 → [[06_Insight]]
                    │      ├─ 一本书 → [[23_Book]]
                    │      └─ 领域 → [[07_System]]
                    │
                    └─ 否 → 问题4: 这是实体吗？
                             ├─ 人 → [[05_Persona]]
                             ├─ 组织 → [[24_Organization]]
                             ├─ 群体 → [[25_Community]]
                             ├─ 物 → [[04_Asset]]
                             └─ 其他 → 查看完整列表
```

## ❓ 还是不确定？

### 经验法则
1. **从简单开始**: 不确定就用 [[01_Seed]]，之后再转化
2. **一物多用**: 同一个事物可以用多个模板记录不同侧面
3. **灵活调整**: 模板是工具，不是枷锁
4. **定期回顾**: 每月检查一次是否用对了模板

**案例1: 学Python**
