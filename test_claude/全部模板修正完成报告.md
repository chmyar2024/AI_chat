## 🎯 统一逻辑框架应用

### 所有模板的标准结构

```markdown
---
[YAML元数据]
---

> [!NOTE] 核心原则
> 目的 / 触发 / 规则

> [!TIP] 边界说明 (如有)

---

## 1️⃣ 定义与现状
## 2️⃣ 价值与目标  
## 3️⃣ 方法与执行
## 4️⃣ 连接与关系
## 5️⃣ 追踪与演化
## 6️⃣ 反馈与优化

---

## ⚡ 快速工具
- 快速参考
- 检查清单
- 使用提示
```

### 按模板类型的逻辑差异

**输入型** (Seed, Intuition, Daily):
→ 捕捉 → 反应 → 连接 → 处理

**加工型** (Insight, Action, Project, Skill):
→ 定义 → 现状 → 执行 → 资源 → 追踪 → 反馈

**关系型** (Persona, Organization, Community):
→ 画像 → 关系 → 价值 → 互动 → 维护 → 发展

**反馈型** (Loop, Meta, Recall):
→ 事实 → 分析 → 学习 → 更新

**战略型** (Vision, Heart, Scale, Timeflow):
→ 愿景 → 现状 → 路径 → 连接 → 校准
